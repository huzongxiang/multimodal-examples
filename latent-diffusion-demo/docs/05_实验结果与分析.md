# 5. 实验结果与分析

完成前面的数据准备、模型搭建与训练推理代码之后，本章回到结果本身：这套系统最终产生了什么样的输出？这些结果是否能够说明模型已经学到了文本到图像生成的基本能力？在当前这套小规模配置下，模型的表现又有哪些值得注意的特点？

本章不追求对结果做过度包装，而是围绕本次真实运行得到的日志、曲线、重建图和采样图，给出一个面向实践的分析。

## 5.1 本次实验设置

为了兼顾单卡环境下的训练速度与结果观察，这里最终采用的是一个“半量数据 + 中等训练强度”的方案。具体设置如下：

- 数据集：`small_ldt` 的 `mj` 预处理特征半量子集
- 训练样本数：`299,898`
- 模型配置：
  - `embed_dim = 768`
  - `n_layers = 12`
  - `patch_size = 2`
  - `image_size = 32`
- 训练配置：
  - `batch_size = 64`
  - `epoch = 5`
  - `recon` 每 `1000 step` 保存一次
  - `sample` 每 `3000 step` 保存一次
  - `checkpoint` 每 `3000 step` 保存一次
  - 不启用早停

这组配置的目标并不是追求完全收敛，而是在可控的训练时间内，清楚展示模型从随机初始化逐步学到图像结构和文本条件的过程。

第二，随着训练继续进行，loss 进入缓慢下降阶段，但并没有在第一个 epoch 内完全停滞。虽然 step loss 在单个 epoch 的后半段会出现平台化现象，但从完整 epoch 统计来看，模型仍然在持续进步。

从曲线图中可以看到：

- `step loss` 在最开始迅速下降，随后进入平稳波动区间；

这意味着在当前训练轮数下，模型已经学到了有效信息，但还没有到“完全收敛”的阶段。它已经从“完全随机生成”进入到了“能够稳定形成类别和结构”的阶段，但仍然有继续优化的空间。

从重建图可以观察到一个非常直观的现象：在训练后期，预测结果已经能够较好地保留原始图像的主体结构、颜色布局和大致语义类型。例如角色、室内产品图、徽标、人物和图案类样本，在下排预测图中都能找到明显对应关系。

## 5.5 使用最佳权重进行文本采样

训练结束之后，又使用最佳权重 `best_model.pth` 进行了一轮独立的文本提示词采样，用来测试模型在真实 prompt 条件下的输出情况。相关文件保存在：

- [prompt_samples_best_simple](/D:/codex/multimodal-examples/latent-diffusion-demo/outputs/small_ldt_half_formal_e5_bs64/prompt_samples_best_simple:1)
- 总览图：[prompt_gallery.png](/D:/codex/multimodal-examples/latent-diffusion-demo/outputs/small_ldt_half_formal_e5_bs64/prompt_samples_best_simple/prompt_gallery.png:1)
- 提示词清单：[prompts.json](/D:/codex/multimodal-examples/latent-diffusion-demo/outputs/small_ldt_half_formal_e5_bs64/prompt_samples_best_simple/prompts.json:1)

这一轮采样共使用了 `8` 个提示词，每个提示词生成 `4` 张图像，采样参数为 `class_guidance = 5.0`、`n_iter = 20`。提示词保持为较短、较直接的形式，包括：

- `woman portrait`
- `cyberpunk city`
- `living room`
- `fantasy character`
- `skincare product`
- `sloth sticker`
- `ocean waves`
- `logo symbol`

从最终采样结果来看，可以得到以下几个判断。

### 5.5.1 场景类提示词表现更稳定

诸如 `cyberpunk city`、`living room`、`ocean waves` 这类场景型或环境型提示词，生成结果相对更稳定。图像中往往能够形成较明确的空间结构、颜色风格和场景氛围。

这说明当前模型对“大块面场景语义”和“整体风格方向”的掌握要好于对精细对象细节的掌握。

### 5.5.2 风格和类别感已经建立起来

诸如 `fantasy character`、`logo symbol`、`sloth sticker` 这类提示词，虽然细节还不够完美，但模型已经能够输出与目标类别明显相关的图像。它已经不只是输出随机纹理，而是在向特定视觉类别聚合。

这恰恰体现了文本条件的作用：模型已经学会根据不同的文本 embedding 走向不同的视觉分布。

### 5.5.3 人物和产品图仍然偏弱

`woman portrait` 和 `skincare product` 这类提示词的结果仍然比较模糊，尤其在人脸结构和产品外形上容易出现形变、融化感或局部不稳定的问题。

这说明在当前训练轮数和数据配置下，模型还没有充分掌握更细粒度、更高精度的结构控制能力。一个模型“能生成像样的图”和“能稳定生成高质量图”之间，仍然存在明显差距。

## 5.6 训练成本与资源占用

从本次实验记录来看，在 `299,898` 条样本、`batch_size = 64`、`5 epoch` 配置下：

- 总 step 数为 `23430`
- 最后一个 epoch 用时约为 `1871.38` 秒，约 `31.2` 分钟
- `5` 个 epoch 总计约为 `9397.87` 秒，约 `2.61` 小时
- GPU 峰值显存约为 `10160.86 MB`
- 共保存 `7` 个阶段性 checkpoint，另有 `best_model.pth` 与 `last_checkpoint.pth`

这说明在 12GB 显存环境下，这套基于 Transformer 的 Latent Diffusion 配置是可以实际跑通的。经过适当裁剪之后，它能够在消费级单卡环境中完成训练、验证与采样。

## 5.7 本节小结

综合本次实验结果，可以得出以下结论：

- 模型在当前配置下已经具备了基本的文本到图像生成能力；
- 从损失曲线、重建图和采样图来看，训练过程是稳定且有效的；
- 模型已经能够学习到场景、风格和大类语义，但在人物细节、产品结构和高精度一致性方面仍有不足；
- 在 12GB 显存环境中，通过预处理、模型裁剪和合适的训练策略，可以让一个真实的 Latent Diffusion 项目顺利跑通。

因此，本章结果最重要的意义，并不只是“生成了几张图”，而是验证了这样一件事：前面章节介绍的多模态生成模型原理，经过合理的工程组织和配置裁剪之后，确实可以在真实环境中落成一个可运行、可观察、可分析的系统。
