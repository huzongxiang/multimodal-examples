# 9.2.1 实验背景与目标

文本生成图像系统真正值得观察的，并不只是最终输出的几张样图，而是文本条件如何进入生成过程、扩散模型如何在潜空间中逐步组织图像结构，以及训练中哪些中间结果最能反映模型正在学习什么。这里讨论的正是这样一个经过裁剪的 Latent Diffusion 案例：它能够在单卡环境中完成训练、观察和推理，重点是把主链讲清楚，而不是把有限规模的结果包装成成熟系统。

## 1. 案例对象

这一案例把文本编码、潜空间表示、条件去噪和图像解码拆开观察。与“输入一句提示词、等待模型输出结果”的黑箱式体验相比，这种写法更容易看清系统真正由哪些部分构成：

- 文本编码器负责把提示词转成条件向量；
- 图像编码器负责把原始图像压缩到潜空间；
- 条件去噪器在潜空间中学习从噪声恢复图像结构；
- 图像解码器再把潜变量还原成最终可见图像。

也正因为系统是分层组织的，这一案例不仅能展示最终生成图像，还能展示训练过程中更有解释力的中间结果，例如重构图和阶段性采样图。与最后一批样张相比，这类中间结果更能反映模型如何从“几乎不会生成”逐步过渡到“开始具备场景和类别感”。

## 2. 为什么选择 Latent Diffusion

对于文本生成图像任务，如果直接在像素空间中做扩散建模，训练和推理成本通常都很高。Latent Diffusion 的基本思路，是先把图像映射到更紧凑的潜空间，再在这个潜空间中完成加噪、去噪与采样。这样做的直接好处有两点：

- 计算成本和显存压力明显下降，更适合单卡和小规模实验环境；
- 模型主线更容易拆解，便于把文本条件、潜变量建模和解码重建分别讲清楚。

在本项目中，原始图像并不是直接作为高分辨率像素进入扩散模型，而是先经过 VAE 编码为 latent；文本也不是以字符串形式直接送入 denoiser，而是先经过 CLIP 文本编码器转成 embedding。真正被训练的部分，是一个以文本 embedding 为条件的潜空间去噪模型。这一结构既保留了当代文本生成图像系统的核心范式，又足够克制，适合作为图书中的实践案例。

## 3. Demo 定位

这个 Demo 采用的是“可运行、可观察、可解释”的取舍。它不追求超大规模数据训练、严格 benchmark 对比、生产级采样质量或真实部署鲁棒性，而把重点放在以下几件事上：

- 文本条件、潜空间表示和扩散去噪是如何连成一条完整主链的；
- 训练过程中，loss、重构图和采样图是否呈现出一致的改善趋势；
- 训练完成后，最佳权重在小规模 prompt 集上的采样结果能说明什么；
- 当前结果的能力边界到底落在什么位置。

因此，有限轮数、小规模 prompt 和现有视觉失真都不应被包装成“已经足够真实使用”的结论。更准确地说，这一案例的目标，是通过一个可控项目展示文本生成图像系统从训练到推理的关键现象，而不是把它直接当成可部署产品。

## 4. 观察重点

这一案例的观察重点集中在三条主线上。

第一条主线是系统结构。模型结构、数据准备与训练入口如何协同，决定了一个 Latent Diffusion 项目能否真正组织成可运行的工程闭环。

第二条主线是训练演化过程。这里不只保留最后一次采样图，而是重点观察训练迭代增加时三类信号如何共同变化：loss 如何下降，重构图如何从粗糙模糊逐步保留主体结构，阶段性采样图如何从接近噪声和色块逐步变得更有场景轮廓和类别感。

第三条主线是训练完成后的推理采样。最佳权重在一组简洁提示词上的表现，可以直接说明哪些类别已经形成较稳定输出，哪些类别仍然存在明显结构缺陷。把这三条主线放在一起，模型原理、训练过程和最终生成效果也就能够在同一个连续视角下被理解。
