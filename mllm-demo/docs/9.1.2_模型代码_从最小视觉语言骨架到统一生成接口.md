# 9.1.2 模型代码：从最小视觉语言骨架到统一生成接口

统一生成接口并不是抽象概念，而是由模型、数据、训练、评估和推理几个模块共同支撑起来的。代码层最值得把握的，不是逐行实现细节，而是这些模块如何围绕同一条视觉前缀到文本生成的主线协同工作，使后续各阶段实验始终落在同一骨架之上。

## 1. 核心模块分类

系统的核心实现由模型、数据、训练、评估和推理几个模块共同构成。各模块并不是彼此孤立存在，而是围绕统一的多模态生成接口分工协作：模型定义模块负责构造视觉—语言骨架，数据模块负责组织训练样本，训练模块负责执行阶段化训练与结果落盘，评估模块负责计算任务指标并保存预测，推理模块负责加载权重并生成最终输出。其分类关系如表9.1.2所示。

表9.1.2 核心模块分类

| 类别 | 模块 | 主要职责 | 主要输入 | 主要输出 |
| --- | --- | --- | --- | --- |
| 模型定义模块 | `model.py` | 定义视觉编码器、映射层、语言模型以及统一前向路径 | 图像张量、文本 token、注意力掩码、标签 | 训练损失、生成文本、阶段权重 |
| 数据组织模块 | `data.py` | 读取 JSONL 样本，完成图像处理、文本编码与标签构造 | JSONL 记录、图像文件、分词器、图像预处理器 | `pixel_values`、`input_ids`、`attention_mask`、`labels` |
| 训练执行模块 | `train.py` | 统一管理训练参数、断点恢复、日志记录、损失曲线、早停和最终保存 | 训练数据、初始权重、训练超参数 | checkpoint、metrics、loss 曲线、最终模型 |
| 指标计算模块 | `eval_utils.py` | 统一实现图像描述、目标检测与视觉问答的指标计算与预测保存 | 模型输出、目标文本、评估数据 | 各任务指标、评估样本结果 |
| 评估入口模块 | `eval.py` | 加载模型并调用评估逻辑，生成阶段评估文件 | checkpoint、评估数据、生成参数 | `eval_metrics.json`、`predictions.jsonl` |
| 推理入口模块 | `infer.py` | 面向单样本推理，加载模型并执行生成 | checkpoint、图像路径、提示词 | 单条推理输出文本 |

以上模块共同构成了一套完整的运行链路：`data.py` 负责把样本整理成统一的训练输入，`model.py` 负责完成图像与文本的联合建模，`train.py` 负责阶段化训练与结果保存，`eval_utils.py` 与 `eval.py` 负责阶段评估，`infer.py` 则提供面向单样本的统一推理接口。各模块的分工关系如下。

从模型结构角度看，系统只有一套统一主体，即 `MiniVLM`。从配置角度看，系统包含两组底座规格，如表9.1.3所示。

表9.1.3 两组底座配置

| 配置名称 | 视觉编码器 | 语言模型 | 对应阶段 | 作用 |
| --- | --- | --- | --- | --- |
| 主分支配置 | `google/vit-large-patch16-224` | `HuggingFaceTB/SmolLM-360M` | Part 1、Part 2、Part 3 | 支撑图像描述、目标检测与视觉问答主链路 |
| 小分支配置 | `google/vit-base-patch16-224` | `HuggingFaceTB/SmolLM-135M` | Part 4 预热分支、Part 4 | 支撑多任务小模型分支 |

因此，这里的“模型”需要区分两个层次来理解：如果按结构统计，只有一套 `MiniVLM`；如果按阶段配置统计，则有两组不同规模的底座组合。这里所说的阶段差异，主要是指这两组配置与不同数据任务之间的对应关系，而不是完全不同的模型结构。

## 2. 模型定义模块：`model.py`

模型定义模块负责建立系统的主体结构，其核心不是引入复杂的跨模态交互网络，而是通过一个映射层将视觉特征压缩为语言模型上下文中的前缀表示。模型由三个部分组成：视觉编码器、映射层和语言模型。视觉编码器负责提取图像 patch 表示，映射层负责把视觉表示投影到语言模型隐藏维度，语言模型则在统一上下文中完成训练和生成。

这一模块包含三个核心对象。

1. `ModelConfig`：负责描述模型名称与视觉分支是否冻结。
2. `VisionProjector`：负责实现视觉特征到语言嵌入空间的非线性投影。
3. `MiniVLM`：负责统一封装图像编码、前向传播、自回归生成以及权重保存和恢复。

完整模型代码如下。

```python
from __future__ import annotations

import json
from dataclasses import dataclass
from pathlib import Path

import torch
import torch.nn as nn
import torch.nn.functional as F
from transformers import AutoModelForCausalLM, AutoTokenizer, ViTImageProcessor, ViTModel


@dataclass
class ModelConfig:
    vision_model_name: str
    lm_model_name: str
    freeze_vision: bool = True


class VisionProjector(nn.Module):
    def __init__(self, vision_dim: int, language_dim: int):
        super().__init__()
        self.projection = nn.Sequential(
            nn.Linear(vision_dim, language_dim),
            nn.GELU(),
            nn.LayerNorm(language_dim),
            nn.Linear(language_dim, language_dim),
        )

    def forward(self, vision_features: torch.Tensor) -> torch.Tensor:
        return self.projection(vision_features)


class MiniVLM(nn.Module):
    def __init__(self, config: ModelConfig):
        super().__init__()
        self.config = config

        self.vision_encoder = ViTModel.from_pretrained(config.vision_model_name)
        self.language_model = AutoModelForCausalLM.from_pretrained(config.lm_model_name)
        self.tokenizer = AutoTokenizer.from_pretrained(config.lm_model_name)
        self.image_processor = ViTImageProcessor.from_pretrained(config.vision_model_name)

        if self.tokenizer.pad_token is None:
            self.tokenizer.pad_token = self.tokenizer.eos_token

        self.vision_dim = self.vision_encoder.config.hidden_size
        self.language_dim = self.language_model.config.hidden_size
        self.projector = VisionProjector(self.vision_dim, self.language_dim)

        if config.freeze_vision:
            for param in self.vision_encoder.parameters():
                param.requires_grad = False

    def encode_image(self, pixel_values: torch.Tensor) -> torch.Tensor:
        if self.config.freeze_vision:
            with torch.no_grad():
                vision_outputs = self.vision_encoder(pixel_values=pixel_values)
        else:
            vision_outputs = self.vision_encoder(pixel_values=pixel_values)

        image_features = vision_outputs.last_hidden_state
        return self.projector(image_features)

    def forward(
        self,
        pixel_values: torch.Tensor,
        input_ids: torch.Tensor,
        attention_mask: torch.Tensor,
        labels: torch.Tensor | None = None,
    ):
        batch_size = pixel_values.shape[0]
        image_embeds = self.encode_image(pixel_values)
        num_image_tokens = image_embeds.shape[1]

        embedding_layer = self.language_model.get_input_embeddings()
        lm_dtype = embedding_layer.weight.dtype
        image_embeds = image_embeds.to(dtype=lm_dtype)
        text_embeds = embedding_layer(input_ids).to(dtype=lm_dtype)
        combined_embeds = torch.cat([image_embeds, text_embeds], dim=1)

        image_attention = torch.ones(
            (batch_size, num_image_tokens),
            dtype=attention_mask.dtype,
            device=attention_mask.device,
        )
        combined_attention = torch.cat([image_attention, attention_mask], dim=1)

        if labels is not None:
            image_labels = torch.full(
                (batch_size, num_image_tokens),
                fill_value=-100,
                dtype=labels.dtype,
                device=labels.device,
            )
            combined_labels = torch.cat([image_labels, labels], dim=1)
        else:
            combined_labels = None

        return self.language_model(
            inputs_embeds=combined_embeds,
            attention_mask=combined_attention,
            labels=combined_labels,
            return_dict=True,
        )

    @torch.no_grad()
    def generate(
        self,
        pixel_values: torch.Tensor,
        prompt: str = "",
        max_new_tokens: int = 50,
        temperature: float = 0.7,
        do_sample: bool = True,
    ) -> str:
        self.eval()
        image_embeds = self.encode_image(pixel_values)
        embedding_layer = self.language_model.get_input_embeddings()
        lm_dtype = embedding_layer.weight.dtype
        image_embeds = image_embeds.to(dtype=lm_dtype)

        if prompt:
            prompt_ids = self.tokenizer.encode(prompt, return_tensors="pt").to(pixel_values.device)
        else:
            bos_token_id = self.tokenizer.bos_token_id
            if bos_token_id is None:
                bos_token_id = self.tokenizer.eos_token_id
            prompt_ids = torch.tensor([[bos_token_id]], device=pixel_values.device)

        generated_ids = prompt_ids.clone()

        for _ in range(max_new_tokens):
            current_embeds = embedding_layer(generated_ids).to(dtype=lm_dtype)
            full_embeds = torch.cat([image_embeds, current_embeds], dim=1)
            outputs = self.language_model(inputs_embeds=full_embeds)
            next_token_logits = outputs.logits[:, -1, :]

            if do_sample:
                probs = F.softmax(next_token_logits / max(temperature, 1e-5), dim=-1)
                next_token = torch.multinomial(probs, num_samples=1)
            else:
                next_token = next_token_logits.argmax(dim=-1, keepdim=True)

            generated_ids = torch.cat([generated_ids, next_token], dim=1)
            if next_token.item() == self.tokenizer.eos_token_id:
                break

        return self.tokenizer.decode(generated_ids[0], skip_special_tokens=True)

    def save_pretrained(self, output_dir: str | Path) -> None:
        output_path = Path(output_dir)
        output_path.mkdir(parents=True, exist_ok=True)

        checkpoint = {
            "config": {
                "vision_model_name": self.config.vision_model_name,
                "lm_model_name": self.config.lm_model_name,
                "vision_dim": self.vision_dim,
                "language_dim": self.language_dim,
                "freeze_vision": self.config.freeze_vision,
            },
            "projector_state_dict": self.projector.state_dict(),
            "language_model_state_dict": self.language_model.state_dict(),
        }

        torch.save(self.projector.state_dict(), output_path / "projector.pt")
        torch.save(checkpoint, output_path / "mini_vlm_full.pt")
        self.tokenizer.save_pretrained(output_path / "tokenizer")
        self.image_processor.save_pretrained(output_path / "image_processor")

        with (output_path / "config.json").open("w", encoding="utf-8") as handle:
            json.dump(checkpoint["config"], handle, indent=2)

    @classmethod
    def from_pretrained(cls, checkpoint_dir: str | Path):
        checkpoint_path = Path(checkpoint_dir)
        checkpoint = torch.load(checkpoint_path / "mini_vlm_full.pt", map_location="cpu")
        config = ModelConfig(
            vision_model_name=checkpoint["config"]["vision_model_name"],
            lm_model_name=checkpoint["config"]["lm_model_name"],
            freeze_vision=checkpoint["config"].get("freeze_vision", True),
        )

        model = cls(config)
        model.projector.load_state_dict(checkpoint["projector_state_dict"])
        model.language_model.load_state_dict(checkpoint["language_model_state_dict"])
        model.tokenizer = AutoTokenizer.from_pretrained(checkpoint_path / "tokenizer")
        model.image_processor = ViTImageProcessor.from_pretrained(checkpoint_path / "image_processor")

        if model.tokenizer.pad_token is None:
            model.tokenizer.pad_token = model.tokenizer.eos_token

        return model
```

从功能上看，`MiniVLM` 内部包含三条关键路径。第一条路径是图像编码路径，即 `encode_image()`；第二条路径是训练路径，即 `forward()` 中的图像前缀与文本嵌入拼接；第三条路径是推理路径，即 `generate()` 中的自回归生成。三条路径共享同一模型主体，因此模型在不同阶段无须改变主体结构，变化只体现在提示模板与目标文本的组织方式上。

## 3. 数据组织模块：`data.py`

数据组织模块负责把 JSONL 记录转换为训练阶段可直接消费的输入张量。其核心任务包括：读取图像、选择提示模板、选择目标文本、构造训练文本、编码 token、生成注意力掩码和标签。这样一来，图像描述、目标检测、视觉问答和多任务混合数据就能够在同一数据接口下完成加载。

这一模块包含四个关键部分。

1. `load_jsonl_records()`：负责读取 JSONL 文件。
2. `select_prompt()`：负责从提示模板中选择当前使用的提示词。
3. `select_target()`：负责从图像描述列表或 `target` 字段中选择目标文本。
4. `JsonlVisionLanguageDataset`：负责把图像和文本统一组织为模型输入。

完整数据代码如下。

```python
from __future__ import annotations

import json
import random
from pathlib import Path
from typing import Any

from PIL import Image
from torch.utils.data import Dataset


def load_jsonl_records(jsonl_path: str | Path) -> list[dict[str, Any]]:
    path = Path(jsonl_path)
    if not path.exists():
        raise FileNotFoundError(f"Dataset not found: {path}")

    records: list[dict[str, Any]] = []
    with path.open("r", encoding="utf-8") as handle:
        for line in handle:
            line = line.strip()
            if line:
                records.append(json.loads(line))
    return records


def select_prompt(record: dict[str, Any], deterministic: bool = False) -> str:
    prompt_options = record.get("prompt_options")
    if prompt_options:
        return prompt_options[0] if deterministic else random.choice(prompt_options)
    return record.get("prompt", "")


def select_target(record: dict[str, Any], deterministic: bool = False) -> str:
    if "captions" in record:
        captions = record["captions"]
        return captions[0] if deterministic else random.choice(captions)
    return record["target"]


class JsonlVisionLanguageDataset(Dataset):
    """JSONL dataset that mirrors the original notebook task formatting."""

    def __init__(
        self,
        jsonl_path: str | Path,
        image_processor,
        tokenizer,
        max_length: int = 256,
    ):
        self.jsonl_path = Path(jsonl_path)
        if not self.jsonl_path.exists():
            raise FileNotFoundError(f"Dataset not found: {self.jsonl_path}")

        self.image_processor = image_processor
        self.tokenizer = tokenizer
        self.max_length = max_length
        self.records = load_jsonl_records(self.jsonl_path)

    def __len__(self) -> int:
        return len(self.records)

    def __getitem__(self, index: int) -> dict[str, Any]:
        record = self.records[index]
        image_path = Path(record["image"])
        if not image_path.exists():
            raise FileNotFoundError(f"Image not found: {image_path}")

        image = Image.open(image_path).convert("RGB")
        pixel_values = self.image_processor(image, return_tensors="pt").pixel_values.squeeze(0)

        prompt = select_prompt(record, deterministic=False)
        target = select_target(record, deterministic=False)
        append_eos = record.get("append_eos", bool(prompt))
        mask_prompt_loss = record.get("mask_prompt_loss", bool(prompt))

        text = f"{prompt} {target}".strip() if prompt else target
        if append_eos and self.tokenizer.eos_token is not None:
            text = f"{text}{self.tokenizer.eos_token}"

        encoding = self.tokenizer(
            text,
            max_length=self.max_length,
            padding="max_length",
            truncation=True,
            return_tensors="pt",
        )
        input_ids = encoding["input_ids"].squeeze(0)
        attention_mask = encoding["attention_mask"].squeeze(0)

        labels = input_ids.clone()
        if mask_prompt_loss and prompt:
            prompt_tokens = self.tokenizer.encode(prompt, add_special_tokens=False)
            prompt_len = min(len(prompt_tokens), labels.numel())
            labels[:prompt_len] = -100
        labels[attention_mask == 0] = -100

        return {
            "pixel_values": pixel_values,
            "input_ids": input_ids,
            "attention_mask": attention_mask,
            "labels": labels,
            "task": record.get("task", "caption"),
            "prompt": prompt,
            "target": target,
            "image_path": str(image_path),
        }
```

这一模块的核心意义在于把不同任务统一为同一种数据结构。无论是自然语言图像描述、JSON 风格检测结果，还是问答任务中的短答案，最终都被组织为一段目标文本，模型总是接收图像和文本序列，而不需要针对不同任务切换不同的数据加载器。

## 4. 训练执行模块：`train.py`

训练执行模块是整个系统的流程控制中心。它并不直接改变模型结构，而是负责把模型、数据、优化器、日志系统、评估逻辑和阶段化产物组织成一个完整的训练闭环。该模块的功能可以分为五类。

**1）参数管理**

`build_parser()` 定义了统一训练入口所需的全部命令行参数，包括：

- 数据输入项：`--train-jsonl`、`--eval-jsonl`
- 结果输出项：`--output-dir`、`--checkpoint-path`、`--metrics-path`
- 初始化来源：`--init-from`
- 模型规格：`--vision-model`、`--lm-model`
- 训练超参数：`--epochs`、`--batch-size`、`--lr`、`--max-length`
- 训练控制项：`--seed`、`--device`、`--early-stop-patience`、`--early-stop-min-delta`
- 视觉分支设置：`--no-freeze-vision`

这样一来，不同阶段的训练差异就不需要写进模型代码，而是通过统一入口参数进行控制。

**2）训练准备**

训练开始前，模块主要完成以下工作。

1. 通过 `set_seed()` 固定随机种子。
2. 构造 `ModelConfig` 并实例化 `MiniVLM`。
3. 如果指定 `--init-from`，则从上一阶段权重继续初始化，恢复 `projector` 和语言模型参数。
4. 使用 `JsonlVisionLanguageDataset` 和 `DataLoader` 构建训练数据迭代器。
5. 创建 `AdamW` 优化器，并根据需要恢复断点状态。

这一过程保证了阶段之间可以串联训练，同时又不会破坏统一模型入口。

**3）训练循环**

训练循环位于 `main()` 中，其内部逻辑包括：

- 从 batch 中读取 `pixel_values`、`input_ids`、`attention_mask` 和 `labels`
- 执行前向计算并得到 `loss`
- 反向传播、梯度裁剪和参数更新
- 逐 step 打印损失并将结果写入 `metrics.jsonl`
- 逐 epoch 汇总平均损失并写入 `losses.json`
- 在每个 epoch 结束后刷新 `loss_curve.png` 和 `run_summary.json`

其中，训练监控指标统一采用 epoch 平均损失。若设置了早停参数，系统会根据 `best_metric`、`stale_epochs` 和 `early_stop_min_delta` 判断是否结束训练。

**4）阶段保存与断点恢复**

训练模块内置了完整的断点与结果保存机制。其主要函数包括：

- `save_epoch_checkpoint()`：保存当前 epoch、loss 历史、模型状态和优化器状态。
- `append_jsonl()`：向 `metrics.jsonl` 持续追加 step 或 epoch 级记录。
- `write_run_config()`：保存当前阶段的配置快照。
- `write_run_summary()`：生成阶段摘要文件，汇总最佳 epoch、最终损失、总 step 数和主要产物信息。

因此，训练过程不仅保留最终权重，也保留了一套完整的过程性结果文件。

**5）曲线绘制与输出产物**

为了保证结果可直接用于后续分析，训练模块还负责生成损失曲线与运行摘要。相关函数为：

- `plot_loss_curves()`：从 `metrics.jsonl` 中读取 step 级和 epoch 级损失。
- `render_loss_plot_with_pil()`：使用 PIL 绘制 `loss_curve.png`。

一个阶段训练结束后，通常会形成如下核心产物：

- `mini_vlm_full.pt`：最终完整模型权重
- `training_checkpoint.pt`：可恢复训练的断点文件
- `metrics.jsonl`：逐 step 与逐 epoch 指标记录
- `losses.json`：epoch 平均损失序列
- `loss_curve.png`：损失曲线图
- `run_config.json`：运行配置
- `run_summary.json`：阶段摘要

因此，`train.py` 的作用不仅是执行训练，更在于把阶段训练组织成一套可以追踪、可以恢复、可以分析的完整实验过程。

## 5. 评估与指标模块：`eval_utils.py` 与 `eval.py`

评估相关功能由两个模块共同组成：`eval_utils.py` 负责具体的指标计算与结果保存，`eval.py` 负责命令行入口与文件输出。

**1）指标计算模块：`eval_utils.py`**

`eval_utils.py` 中包含一组围绕文本生成结果设计的评估函数，具体如下。

| 函数 | 作用 |
| --- | --- |
| `normalize_text()` | 对文本统一做小写、空白压缩和符号清理 |
| `strip_prompt_prefix()` | 从模型输出中剥离提示词前缀，提取有效生成内容 |
| `token_f1()` | 计算预测与参考文本之间的 token 级 F1 |
| `best_caption_scores()` | 在多参考图像描述中选取最优 exact match 与 token F1 |
| `safe_json_loads()` | 从文本中提取 JSON 对象并安全解析 |
| `bbox_iou()` | 计算两个边界框之间的 IoU |
| `detection_metrics()` | 计算检测任务中的 JSON 可解析性、目标数准确率、标签准确率和 IoU |
| `evaluate_records()` | 统一执行逐样本推理并汇总各任务指标 |
| `save_predictions()` | 保存逐样本预测结果 |
| `save_metrics()` | 保存评估摘要 |

其中最关键的函数是 `evaluate_records()`。它会逐条读取 JSONL 样本，调用模型推理，按任务类型分别计算图像描述、视觉问答和目标检测的指标，并最终返回一个阶段级汇总结果与逐样本预测列表。这样一来，不同任务就可以共享同一个评估入口，而不需要为每一阶段单独设计评估流程。

**2）评估入口模块：`eval.py`**

`eval.py` 是评估入口模块，其职责是：

1. 接收模型恢复位置和评估数据。
2. 调用 `MiniVLM.from_pretrained()` 恢复模型。
3. 调用 `evaluate_records()` 计算指标。
4. 将结果写入 `eval_metrics.json` 和 `predictions.jsonl`。

这种实现方式使评估与训练阶段解耦。训练可以在阶段结束后统一调用评估，也可以在任意时刻单独对某个 checkpoint 执行离线评估。

## 6. 推理入口模块：`infer.py`

推理入口模块负责单样本推理。该模块的职责并不复杂，但它把多模态系统的实际使用方式固定为统一接口：加载模型、读取图像、构造提示词、调用 `generate()`、输出文本。完整推理代码如下。

```python
from __future__ import annotations

import argparse

import torch
from PIL import Image

from .model import MiniVLM


def build_parser() -> argparse.ArgumentParser:
    parser = argparse.ArgumentParser(description="Run inference with a saved MiniVLM checkpoint.")
    parser.add_argument("--checkpoint-dir", type=str, required=True)
    parser.add_argument("--image-path", type=str, required=True)
    parser.add_argument("--prompt", type=str, default="")
    parser.add_argument("--max-new-tokens", type=int, default=50)
    parser.add_argument("--temperature", type=float, default=0.7)
    parser.add_argument("--do-sample", action="store_true")
    parser.add_argument("--device", type=str, default="cuda" if torch.cuda.is_available() else "cpu")
    return parser


def main() -> None:
    args = build_parser().parse_args()
    device = torch.device(args.device)

    model = MiniVLM.from_pretrained(args.checkpoint_dir)
    model.to(device)

    image = Image.open(args.image_path).convert("RGB")
    pixel_values = model.image_processor(image, return_tensors="pt").pixel_values.to(device)
    output = model.generate(
        pixel_values=pixel_values,
        prompt=args.prompt,
        max_new_tokens=args.max_new_tokens,
        temperature=args.temperature,
        do_sample=args.do_sample,
    )
    print(output)


if __name__ == "__main__":
    main()
```

在不同任务下，推理入口保持不变，变化只体现在提示词与结果解释方式上。

- 在图像描述阶段，输出通常按自然语言句子解释。
- 在目标检测阶段，输出需要进一步解析为 JSON 结果。
- 在视觉问答阶段，输出通常是短答案文本。
- 在多任务阶段，同一接口通过不同提示词完成任务切换。

因此，`infer.py` 的意义不在于提供复杂的推理流程，而在于将统一生成接口落实到单样本使用场景中。

## 7. 模块关系说明

从整体结构看，这些核心模块共同构成了一条清晰的执行链路：`data.py` 提供统一样本接口，`model.py` 提供统一多模态骨架，`train.py` 负责阶段训练和产物落盘，`eval_utils.py` 与 `eval.py` 负责阶段评估，`infer.py` 负责最终推理。系统的稳定性和可扩展性并不来自复杂的网络拆分，而来自这些模块在同一生成式接口下的分工协作。

正因为核心模块已经按类别清晰分离，后续各阶段才能在不改动模型主体的前提下，通过切换数据内容、提示模板、初始化权重和训练配置完成能力扩展。也正因为如此，训练、推理与结果分析始终能够落在同一套模块体系上。
