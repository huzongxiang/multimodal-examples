# 9.2.3.4 Part 4：多任务统一——最小 MLLM 的任务切换能力

## 1. 阶段目标

Part 4 的任务是将图像描述、目标检测和视觉问答三类任务统一到同一个模型中，使系统在共享参数的条件下，能够根据不同提示词在不同任务之间切换行为。该阶段的重点不再是证明单一任务能否成立，而是检验统一生成接口是否已经足以支撑多任务系统的基本形态。

与前面三个阶段相比，Part 4 的核心变化不在模型结构，而在训练数据的组织方式和模型分支的选择。该阶段使用更小的一组底座配置，即 `ViT-Base + SmolLM-135M`，并在正式多任务训练前先进行一个图像描述预热步骤，以获得适合该分支的初始化权重。因此，Part 4 观察的重点不再只是单任务效果，而是统一接口下的任务切换能力。

## 2. 数据与任务形式

这一阶段使用 `2400` 条混合样本，其中图像描述、目标检测和视觉问答样本分别为 `1000`、`400` 和 `1000` 条。三类样本被统一组织到同一个训练集合中，模型仍然只接收图像和文本序列，不同任务之间的区分主要依赖提示模板与目标文本格式。

多任务样本的组织可以分为三类。

图像描述样本示例：

```json
{
  "image": "data/repo/part4_multitask/images/caption/00000.jpg",
  "task": "caption",
  "prompt": "Describe this image.",
  "captions": [
    "Boys with their backs against an incoming wave .",
    "Four boys are about to be hit by an approaching wave .",
    "Four people sitting in the path of a wave ."
  ],
  "append_eos": true,
  "mask_prompt_loss": true
}
```

目标检测样本与 Part 2 类似，仍以 JSON 目标为主；视觉问答样本则继续使用 `Question: ... Answer:` 模板。这样一来，多任务训练中的关键变化就不再是数据格式是否统一，而是三类任务是否能够在同一训练过程中共享同一套模型主体。

从模型配置上看，Part 4 使用 `google/vit-base-patch16-224` 作为视觉编码器，使用 `HuggingFaceTB/SmolLM-135M` 作为语言模型，并从 `output/part1_for_part4` 中加载预热权重作为初始化。训练使用 `batch_size=4`、`lr=1e-4`、`max_length=256`，共训练 `50` 个 epoch，总步数为 `30000`。

## 3. 训练与推理主线

Part 4 的训练仍然由统一入口 `train.py` 完成，只是在输入数据和初始化权重上发生变化。训练命令如下。

```powershell
$env:PYTHONPATH = "src"
python -m mllm_demo.train `
  --train-jsonl "data\repo\part4_multitask\train.jsonl" `
  --output-dir "output\part4" `
  --checkpoint-path "output\part4\training_checkpoint.pt" `
  --init-from "output\part1_for_part4" `
  --vision-model "google/vit-base-patch16-224" `
  --lm-model "HuggingFaceTB/SmolLM-135M" `
  --epochs 50 `
  --batch-size 4 `
  --lr 1e-4 `
  --max-length 256 `
  --early-stop-patience 5 `
  --early-stop-min-delta 0.005
```

这一阶段的训练重点不再是让模型适应某一类单独目标文本，而是让同一模型在交替出现的三类任务记录上学会不同的输出模式。图像描述样本要求生成自然语言描述，目标检测样本要求生成结构化 JSON，视觉问答样本要求生成短答案文本。模型主体、损失函数和前向路径全部保持不变，真正承担任务区分作用的是提示模板和目标格式。

推理阶段同样沿用统一接口，只需替换提示词即可切换任务。例如：

- 图像描述：`Describe this image.`
- 目标检测：`What objects are in this image? Output as JSON.`
- 视觉问答：`Question: What is being celebrated? Answer:`

这种推理方式意味着，多任务系统的“多任务”并不是通过三套不同推理函数实现的，而是通过同一生成入口在不同提示词条件下表现出的不同行为实现的。

## 4. 结果分析

从训练结果看，Part 4 完整训练了 `50` 个 epoch，最佳 epoch 为 `47`，最佳 epoch 平均损失约为 `0.3336`，最终 epoch 平均损失约为 `0.3365`。从过程上看，这一阶段已经形成稳定的多任务训练权重，并未在设定轮数内触发早停。

从平衡小样本评估结果看，Part 4 是整个实验链路中最能体现统一接口能力的阶段。对 `18` 条平衡样本进行评估后，三类任务的结果如下。

- 图像描述：`exact match = 0.1667`，`token F1 ≈ 0.7964`
- 目标检测：`json_parse_success = 1.0`，`object_count_accuracy ≈ 0.8333`，`label_accuracy = 1.0`，`bbox_iou ≈ 0.8595`
- 视觉问答：`exact match = 1.0`，`token F1 = 1.0`

需要注意的是，这一评估集来自训练混合集合中的平衡抽样，因此它更适合说明“统一接口下的多任务行为已经形成”，而不宜直接理解为严格的泛化性能结论。

为了更具体地说明这一阶段的输入输出关系，下面分别给出三类任务的样例。

### 4.1 图像描述样例

- 输入图像：`images/caption/00777.jpg`
- 提示词：`Describe this image.`
- 参考描述之一：`Two children are riding horses over sand near a white fence .`
- 模型输出：`Two children are riding horses over sand near a white fence .`

该样例说明，在多任务训练条件下，模型已经能够在统一提示词引导下恢复与图像内容高度一致的自然语言描述。

### 4.2 目标检测样例

- 输入图像：`images/od/00172.jpg`
- 提示词：`What objects are in this image? Output as JSON.`
- 参考输出：`{"objects": [{"label": "person", "bbox": [0.211, 0.402, 0.406, 0.402]}, {"label": "dog", "bbox": [0.514, 0.428, 0.433, 0.381]}]}`
- 模型输出：`{"objects": [{"label": "person", "bbox": [0.211, 0.402, 0.406, 0.402]}, {"label": "dog", "bbox": [0.519, 0.383, 0.381, 0.301]}]}`

该样例说明模型不仅保持了合法 JSON 结构，而且正确恢复了对象类别，边界框也与参考结果较为接近，单样本 IoU 约为 `0.7678`。

### 4.3 视觉问答样例

- 输入图像：`images/vqa/00094.jpg`
- 提示词：`Question: What is being celebrated? Answer:`
- 参考答案：`birthday`
- 模型输出：`birthday`

该样例说明，在统一模型参数下，系统不仅能够描述图像和输出结构化检测结果，还能够在问答模板下切换为短答案生成行为。

### 4.4 多任务切换现象

除上述成功样例之外，Part 4 的输出还显示出典型的任务切换现象。例如，在目标检测任务中，模型可以直接输出 `racoon`、`goat`、`dog` 等对象的 JSON；在视觉问答任务中，又能够对 `What toppings are on the pizza?` 生成 `cheese and oregano`，对 `Is this an unhealthy snack?` 生成 `no`。这说明模型并不是简单记住某一种输出风格，而是在提示词的驱动下切换不同任务模式。

该阶段的意义因此十分明确。首先，它说明统一接口不仅适用于单任务，而是已经能够组织起图像描述、目标检测与视觉问答三类行为。其次，它说明最小多模态系统的关键特征，不仅在于能够同时处理图像和文本，还在于能够在同一套参数下复用同一推理入口完成不同任务。再次，它说明多任务训练在本实验链路中起到了真正的收束作用：前面各阶段形成的能力，在这一阶段被组织到同一个模型之中。

同时，Part 4 的边界也需要明确。由于当前平衡评估样本来自训练混合数据，结果更适合说明“任务切换能力和统一接口已经成立”，而不适合直接表述为严格的泛化验证。换言之，Part 4 的主要结论不是“已经得到一个高性能通用多模态模型”，而是“已经形成一个具有清晰任务切换能力的最小多任务多模态系统”。

因此，Part 4 在整个实验链路中的结论可以概括为：统一生成接口已经能够承载多种视觉语言任务，同一模型能够根据提示词在描述、检测和问答之间切换行为，最小 MLLM 的基本形态由此形成。
