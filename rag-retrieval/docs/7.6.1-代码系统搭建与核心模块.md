# 7.6.1 系统搭建与模块组织

## 1. 工程目标与分层结构

系统采用“本地检索 + 云端生成”的二层结构。本地部分负责 PDF 页面图像化、`ColPali` embedding 计算和 `Milvus Lite` 向量检索；云端部分负责由多模态生成模型读取命中的页面图像，并输出自然语言答案。这样划分后，检索链与生成链的职责边界清楚，系统故障也可以被分段定位。

在这一结构中，各模块承担的职责如下：

1. `PdfManager`：将 PDF 转换为页面图像。
2. `ColpaliManager`：生成页面与问题的多模态表示。
3. `MilvusManager`：保存页面局部向量并执行重排检索。
4. `Middleware`：装配索引与检索流程。
5. `Rag`：对接多模态生成接口。
6. `run_pdf_qa.py`：提供可复现的脚本入口。

上述模块共同构成了案例系统的工程骨架。其中最关键的设计选择是：系统首先把文档视为**页面图像集合**，而不是先将其压平成 OCR 文本块。

## 2. 运行环境与配置约定

系统依赖两类环境要素：Python 依赖和系统依赖。Python 依赖包括 `pdf2image`、`pymilvus`、`colpali_engine`、`python-dotenv` 等；系统依赖则主要是 `poppler`，因为 PDF 转图由其底层工具链完成。

环境准备过程可以写成如下形式：

```bash
cd <project_root>
micromamba create -y -n <env_name> python=3.11
micromamba install -y -n <env_name> -c conda-forge poppler
micromamba run -n <env_name> pip install -r requirements.txt
```

配置文件 `.env` 至少需要提供以下字段：

```dotenv
ZHIPU_API_KEY=<your_api_key>
GLM_MODEL=glm-4.6v-flash
COLPALI_DEVICE=mps
COLPALI_MODEL_PATH=./models/vidore__colpali-v1.2
```

这些字段分别控制两条不同链路：

1. `ZHIPU_API_KEY` 与 `GLM_MODEL` 作用于云端生成链。
2. `COLPALI_DEVICE` 与 `COLPALI_MODEL_PATH` 作用于本地检索链。

这一拆分使系统不必将“本地 embedding 运行条件”和“云端问答接口条件”混为一体。

## 3. 页面处理与多模态表示生成

系统首先通过 `PdfManager` 完成页面图像化处理：

```python
def save_images(self, id, pdf_path, max_pages, pages: list[int] = None) -> list[str]:
    output_folder = f"pages/{id}/"
    images = convert_from_path(pdf_path, poppler_path=_resolve_poppler_path())

    self.clear_and_recreate_dir(output_folder)

    num_page_processed = 0
    for i, image in enumerate(images):
        if max_pages and num_page_processed >= max_pages:
            break
        if pages and i not in pages:
            continue

        full_save_path = f"{output_folder}/page_{i + 1}.png"
        image.save(full_save_path, "PNG")
        num_page_processed += 1

    return [f"{output_folder}/page_{i + 1}.png" for i in range(num_page_processed)]
```

该函数的意义并不止于“PDF 转 PNG”。它决定了系统后续处理的基本对象是页面而不是文本块，因此页面中的表格结构、图表布局和局部空间关系能够被保留下来。

随后，`ColpaliManager` 将页面图像与自然语言问题映射到同一检索空间。设备与模型路径的解析逻辑如下：

```python
def _resolve_device(requested_device: str | None = None) -> str:
    if requested_device:
        return requested_device
    env_device = os.getenv("COLPALI_DEVICE")
    if env_device:
        return env_device
    if torch.cuda.is_available():
        return "cuda"
    if torch.backends.mps.is_available():
        return "mps"
    return "cpu"

def _resolve_model_name(default_model_name: str) -> str:
    configured_path = os.getenv("COLPALI_MODEL_PATH")
    if configured_path:
        model_path = Path(configured_path)
        if not model_path.is_absolute():
            model_path = BASE_DIR / model_path
        if model_path.exists():
            return str(model_path)
    default_local_path = BASE_DIR / "models" / "vidore__colpali-v1.2"
    if default_local_path.exists():
        return str(default_local_path)
    return os.getenv("COLPALI_MODEL_NAME", default_model_name)
```

页面编码与问题编码分别由 `process_images()` 和 `process_text()` 完成：

```python
def process_images(self, image_paths:list[str], batch_size=5):
    self._ensure_model_loaded()
    images = self.get_images(image_paths)
    dataloader = DataLoader(
        dataset=ListDataset[str](images),
        batch_size=batch_size,
        shuffle=False,
        collate_fn=lambda x: self.processor.process_images(x),
    )
    ds: List[torch.Tensor] = []
    for batch_doc in dataloader:
        with torch.no_grad():
            batch_doc = {k: v.to(self.model.device) for k, v in batch_doc.items()}
            embeddings_doc = self.model(**batch_doc)
        ds.extend(list(torch.unbind(embeddings_doc.to(self.device))))
    return [d.float().cpu().numpy() for d in ds]

def process_text(self, texts: list[str]):
    self._ensure_model_loaded()
    dataloader = DataLoader(
        dataset=ListDataset[str](texts),
        batch_size=1,
        shuffle=False,
        collate_fn=lambda x: self.processor.process_queries(x),
    )
    qs: List[torch.Tensor] = []
    for batch_query in dataloader:
        with torch.no_grad():
            batch_query = {k: v.to(self.model.device) for k, v in batch_query.items()}
            embeddings_query = self.model(**batch_query)
        qs.extend(list(torch.unbind(embeddings_query.to(self.device))))
    return [q.float().cpu().numpy() for q in qs]
```

至此，页面与问题虽属不同模态，但已经获得了可比较的表示，这构成了后续多模态检索的前提。

## 4. 向量存储、流程装配与问答接口

在存储层，`MilvusManager` 保存的不是“整页单向量摘要”，而是页面内部局部向量的集合。Schema 设计如下：

```python
schema.add_field(field_name="pk", datatype=DataType.INT64, is_primary=True)
schema.add_field(field_name="vector", datatype=DataType.FLOAT_VECTOR, dim=self.dim)
schema.add_field(field_name="seq_id", datatype=DataType.INT16)
schema.add_field(field_name="doc_id", datatype=DataType.INT64)
schema.add_field(field_name="doc", datatype=DataType.VARCHAR, max_length=65535)
```

对应的页面级重排逻辑为：

```python
results = self.client.search(
    self.collection_name,
    data,
    limit=int(50),
    output_fields=["vector", "seq_id", "doc_id"],
    search_params=search_params,
)

doc_vecs = np.vstack(
    [doc_colbert_vecs[i]["vector"] for i in range(len(doc_colbert_vecs))]
)
score = np.dot(data, doc_vecs.T).max(1).sum()
```

这一公式体现的是 `Late Interaction` 风格的页面级重排：查询的不同部分分别与候选页面内部的局部向量匹配，再聚合成页面总分。

在流程层，`Middleware` 负责把上述能力装配成索引与检索接口：

```python
def index(self, pdf_path: str, id:str, max_pages: int, pages: list[int] = None):
    image_paths = pdf_manager.save_images(id, pdf_path, max_pages)
    colbert_vecs = colpali_manager.process_images(image_paths)

    images_data = [{
        "colbert_vecs": colbert_vecs[i],
        "filepath": image_paths[i]
    } for i in range(len(image_paths))]

    self.milvus_manager.insert_images_data(images_data)
    return image_paths

def search(self, search_queries: list[str]):
    final_res = []
    for query in search_queries:
        query_vec = colpali_manager.process_text([query])[0]
        search_res = self.milvus_manager.search(query_vec, topk=1)
        final_res.append(search_res)
    return final_res
```

在生成层，`Rag` 将命中的页面图像与问题组合为统一输入：

```python
def __get_glm_api_payload(self, query: str, image_paths: List[str]):
    image_payload = []
    for image_path in image_paths:
        base64_image = encode_image(image_path)
        image_payload.append({
            "type": "image_url",
            "image_url": {
                "url": f"data:image/jpeg;base64,{base64_image}"
            }
        })

    payload = {
        "model": os.getenv("GLM_MODEL", "glm-4.6v-flash"),
        "messages": [
            {
                "role": "user",
                "content": [
                    *image_payload,
                    {"type": "text", "text": query}
                ]
            }
        ],
        "max_tokens": 1024
    }
    return payload
```

这说明当前系统的生成接口仍然是最小形态，即“整页图像 + 问题文本”。它已经足以支撑案例运行，但尚未进入区域级证据组织阶段。
