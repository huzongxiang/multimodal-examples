# 9.3 多模态文档检索实战：基于 ColPali、Milvus 与 GLM 的多模态文档问答

本案例围绕一条页面级多模态文档问答主线展开。系统先将 PDF 文档逐页转换为图像，再由 ColPali 对页面与问题生成多向量表示，本地 Milvus 负责完成页面级召回与重排，最后由 GLM 在命中页上执行阅读与回答。与先做 OCR、再把文本切块送入普通文本检索的常见流程相比，这一路线更强调页面作为整体视觉对象的保留，因为在真实文档中，标题与正文的位置关系、表格列对齐、图表标题与图例对应等结构信息，本身就是证据的一部分；只有在检索阶段尽可能保留这种页面结构，后续问答链路才有可能稳定处理表格、图表与复杂版式中的关键信息。

这一案例的重点不在于构建大规模生产级文档问答系统，而在于提供一个可运行、可分析、可复现的最小页面级多模态问答示例。整组内容将围绕页面表示构造、页面级检索、命中页阅读与答案生成逐步展开，重点讨论视觉文档检索为何不同于普通文本检索，Late Interaction 为什么适合承担页面级召回任务，以及检索链与生成链在一个最小系统中如何形成清晰分工。由此，读者看到的将不只是一个能够回答文档问题的原型，而是一条从页面视觉表示到多模态问答闭环的最小实现路径。
