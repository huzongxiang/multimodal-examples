# 7.6 实战：基于 ColPali、Milvus 与 GLM 的多模态文档问答落地

`ColPali + Milvus + GLM` 可以构成一条完整的多模态文档问答主链：PDF 页面先被转换为图像，页面图像随后被编码为可检索表示，相关页面经由本地向量检索召回，命中的页面最终进入多模态生成模型并形成答案。该链路对应的是页面级多模态问答系统的最小可运行形态。

以下记号用于统一表示工程对象：

1. `<project_root>` 表示项目根目录。
2. `<sample_pdf>` 表示用于演示的示例 PDF。
3. `<output_dir>` 表示结果输出目录。
4. `<env_name>` 表示 Python 运行环境名称。

系统搭建、执行主链、结果验证和后续扩展分别在 `7.6.1` 至 `7.6.3` 中展开。
